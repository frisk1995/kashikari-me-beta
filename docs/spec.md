# プロダクト仕様書: かしかり.me

## 概要

「かしかり.me」は、旅行先や日常での「立て替え」「割り勘」を記録し、最終的に「誰が誰にいくら払えばよいか」を自動で精算してくれる iPhone 向けスマホアプリです。

複数人で旅行や飲み会をすると「今回は自分が払っておく」という立て替えが何度も発生し、最後に誰がいくら負担したのか分からなくなりがちです。本アプリは、支払いを記録するだけで割り勘の負担を均等化し、最小回数の送金で精算が完了する組み合わせを提示します。

初期バージョンはオフライン・ローカル保存を前提とし、ログインやアカウント登録なしですぐに使い始められることを重視します。将来的にはログイン機能とサーバ同期に拡張する想定です。

## ターゲットユーザー

- **メインユーザー:** グループ旅行（2〜10人程度）で会計係を担う人。宿泊費・交通費・食事代などをまとめて立て替えることが多い。
- **サブユーザー:** ルームシェア・カップル・サークルなど、日常的に少額の立て替えが繰り返し発生する関係性の人。
- **共通する特徴:** 計算が面倒、割り勘アプリにアカウント登録するのが煩わしい、サッと記録してサッと精算したい。

## 主要機能

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | グループ管理 | 旅行・イベント単位で「グループ」を作成・編集・削除する | Must |
| 2 | メンバー管理 | グループに参加メンバー（名前のみ）を追加・編集・削除する | Must |
| 3 | 支払い記録 | 「誰が」「いくら」「何に」払ったかを記録する | Must |
| 4 | 割り勘対象指定 | 各支払いについて、誰の分の負担かを選択する（全員・一部メンバー） | Must |
| 5 | 精算計算 | グループ内の貸し借りを集計し、最小送金回数の精算案を提示する | Must |
| 6 | 残高表示 | 各メンバーの「払いすぎ／払い足りない」金額を一覧表示する | Must |
| 7 | 支払い一覧・履歴 | グループ内の全支払いを時系列で一覧表示する | Must |
| 8 | 支払いの編集・削除 | 記録済みの支払いを後から修正・削除できる | Must |
| 9 | グループ合計表示 | グループの総支出・1人あたり平均などのサマリーを表示する | Should |
| 10 | カテゴリ分類 | 支払いに「食事」「交通」「宿泊」などのカテゴリを付与する | Should |
| 11 | 不均等割り勘 | 割合や金額を指定して均等以外の割り勘を設定する | Should |
| 12 | 精算完了マーク | 精算済みの送金にチェックを付け、完了状態を管理する | Should |
| 13 | データの永続化 | アプリを閉じても全データがローカルに保持される | Must |
| 14 | 通貨表示 | 金額を日本円フォーマット（カンマ区切り・¥記号）で表示する | Should |
| 15 | 空状態ガイド | データがないときに操作を促すガイド表示を行う | Could |
| 16 | データのエクスポート | グループの精算結果をテキストで共有・コピーできる | Could |

## 画面構成・ユーザーフロー

### 画面一覧

1. **グループ一覧画面（ホーム）**
   - 作成済みグループをカード形式で一覧表示
   - 各カードにグループ名・メンバー数・総支出を表示
   - 「＋新規グループ」ボタン
   - グループが0件のときは空状態ガイドを表示

2. **グループ作成・編集画面**
   - グループ名の入力
   - メンバーの追加・削除（名前入力）
   - 保存／キャンセル

3. **グループ詳細画面（タブ構成）**
   - タブ1「支払い」: 支払い履歴の一覧、「＋支払いを追加」ボタン
   - タブ2「精算」: 各メンバーの残高と精算案（誰が誰にいくら払うか）
   - 画面上部にグループ名・総支出サマリー

4. **支払い追加・編集画面**
   - 金額入力
   - 支払った人（メンバーから選択）
   - 用途（メモ）入力、カテゴリ選択
   - 割り勘対象メンバーの選択（全員選択／個別選択）
   - 保存／削除／キャンセル

5. **精算詳細画面（精算タブ内）**
   - メンバーごとの残高（プラス＝受け取る、マイナス＝支払う）
   - 「AさんがBさんに ¥X,XXX 支払う」形式の精算案リスト
   - 各精算行に「精算済み」チェック
   - 精算結果の共有ボタン

### 主要ユーザーフロー

```
[ホーム]
  → 「＋新規グループ」→ [グループ作成] → メンバー追加 → 保存
  → グループカードをタップ → [グループ詳細/支払いタブ]
       → 「＋支払いを追加」→ [支払い追加] → 金額・支払者・割り勘対象を入力 → 保存
       → 「精算」タブに切替 → [精算詳細] で残高と精算案を確認
       → 精算行の「精算済み」をチェック
```

## スプリント計画

### Sprint 1: グループ管理とデータ永続化の基盤

**実装内容:**
- グループ一覧画面（ホーム）の表示
- グループの新規作成・編集・削除
- グループへのメンバー追加・編集・削除（名前のみ）
- 全データをローカルストレージに永続化し、アプリ再起動後も保持する
- グループが0件のときの空状態ガイド表示

**受け入れ基準:**
- [ ] アプリ起動時にグループ一覧画面（ホーム）が表示される
- [ ] グループが1件もない状態では、空状態ガイド（新規作成を促すメッセージ／ボタン）が表示される
- [ ] 「新規グループ」操作からグループ名を入力して保存すると、一覧にそのグループが追加される
- [ ] グループ作成時にメンバー（2名以上）を追加でき、保存後にメンバー数が一覧カードに反映される
- [ ] 既存グループを編集してグループ名・メンバーを変更すると、変更内容が一覧・詳細に反映される
- [ ] グループを削除すると一覧から消える
- [ ] グループまたはメンバーを追加した後にアプリを再起動（リロード）しても、データが保持されて表示される
- [ ] グループ名が空のまま保存しようとするとエラー／保存不可となり、不正なグループが作成されない

### Sprint 2: 支払いの記録と履歴管理

**実装内容:**
- グループ詳細画面（支払いタブ）の表示
- 支払いの追加（金額・支払者・用途メモ）
- 割り勘対象メンバーの選択（全員選択／個別選択）
- 支払い一覧の時系列表示
- 支払いの編集・削除
- 金額の日本円フォーマット表示（カンマ区切り・¥記号）
- グループ総支出サマリーの表示

**受け入れ基準:**
- [ ] グループ一覧でグループをタップすると、グループ詳細画面（支払いタブ）が開く
- [ ] 「支払いを追加」から金額・支払者・用途を入力して保存すると、支払い一覧に追加される
- [ ] 支払い追加時に割り勘対象メンバーを「全員」または「個別選択」で指定できる
- [ ] 支払い一覧が時系列（新しい順または古い順）で表示される
- [ ] 既存の支払いを開いて金額・支払者・用途・割り勘対象を編集すると、変更が一覧に反映される
- [ ] 支払いを削除すると一覧から消え、総支出サマリーが更新される
- [ ] 金額が「¥1,234」のように円記号とカンマ区切りで表示される
- [ ] 金額が未入力・0・数値以外のとき保存できず、エラーが表示される
- [ ] グループ詳細画面に総支出（全支払いの合計）が正しく表示される
- [ ] 支払いを追加・編集・削除した後にアプリを再起動しても、内容が保持される

### Sprint 3: 精算計算と残高表示

**実装内容:**
- グループ詳細画面（精算タブ）の表示
- 各メンバーの残高計算（払いすぎ＝受け取る／払い足りない＝支払う）
- 最小送金回数の精算案アルゴリズム（誰が誰にいくら払うか）
- 精算案リストの表示
- 1人あたり平均負担額の表示

**受け入れ基準:**
- [ ] グループ詳細画面で「精算」タブに切り替えられる
- [ ] 各メンバーの残高（プラス＝受け取る金額、マイナス＝支払う金額）が一覧表示される
- [ ] 全メンバーの残高の合計が0（ゼロサム）になる
- [ ] 「AさんがBさんに ¥X,XXX 支払う」形式の精算案が表示される
- [ ] 既知の検証ケース（例: 3人で1人が3000円立て替え・全員均等割り）で、精算案が手計算と一致する
- [ ] 支払いが1件もないグループの精算タブでは、残高が全員0または「精算なし」が表示される
- [ ] 支払いを追加・編集・削除すると、精算タブの残高・精算案が即座に再計算される
- [ ] 精算案の送金回数が（メンバー数−1）以下に収まっている

### Sprint 4: 割り勘の高度化と精算管理

**実装内容:**
- カテゴリ分類（食事・交通・宿泊・買い物・その他など）の付与と一覧での表示
- 不均等割り勘（金額指定または比率指定）の設定
- 精算完了マーク（精算済みチェックと完了状態の永続化）
- 支払い一覧のカテゴリ別フィルタまたはカテゴリ表示

**受け入れ基準:**
- [ ] 支払い追加・編集時にカテゴリを選択でき、一覧の各支払いにカテゴリが表示される
- [ ] 均等割り以外（特定メンバーの負担額／比率を変える）の割り勘を設定でき、精算結果に反映される
- [ ] 不均等割り勘を設定した場合でも、全メンバーの残高合計が0になる
- [ ] 精算タブの精算案に「精算済み」チェックを付けられる
- [ ] 「精算済み」にした送金は、見た目（取り消し線・チェック状態など）で完了が区別できる
- [ ] 精算済み状態がアプリ再起動後も保持される

### Sprint 5: サマリー・共有・仕上げ

**実装内容:**
- グループサマリーの拡充（総支出・1人あたり平均・メンバー別支出割合など）
- 精算結果のテキスト共有・コピー機能
- 空状態・エラー状態の文言とガイドの整備
- 全体のUI/UX調整（操作の一貫性、ローディング・確認ダイアログ）

**受け入れ基準:**
- [ ] グループ詳細にサマリー（総支出・1人あたり平均）が表示される
- [ ] 精算結果を共有／コピーする操作ができ、精算案がテキスト形式で出力される
- [ ] グループ削除・支払い削除など破壊的操作の前に確認ダイアログが表示される
- [ ] 各画面の空状態（グループなし・支払いなし・精算なし）で適切なガイドが表示される
- [ ] アプリ全体を通して、作成→記録→精算→共有まで一連の操作がエラーなく完了できる
- [ ] これまでのスプリント（Sprint 1〜4）の機能が引き続き正常に動作する（回帰なし）

## 将来的な拡張機能

以下は本バージョンのスコープ外とし、将来の拡張として想定する。

- **アカウント・ログイン機能:** メールアドレスやSNS連携によるユーザー認証。
- **サーバ同期・クラウド保存:** 複数端末でのデータ共有、機種変更時のデータ引き継ぎ。
- **グループメンバー間のリアルタイム共有:** メンバー各自が自分の端末で支払いを記録し、同一グループに反映。
- **多通貨対応・為替換算:** 海外旅行向けに複数通貨での記録と円換算。
- **レシート読み取り（OCR）:** カメラで撮影した金額の自動入力。
- **プッシュ通知:** 精算依頼・未精算リマインド。
- **決済連携:** 各種送金サービスへのディープリンクによる精算実行。

## 非機能要件

- **プラットフォーム:** iPhone（iOS）対応のスマホアプリとして動作すること。
- **オフライン動作:** ネットワーク接続なしで全機能が利用できること（初期バージョンはローカル保存前提）。
- **データ永続性:** アプリを終了・再起動してもデータが失われないこと。
- **応答性:** 画面遷移・一覧表示・精算計算が体感で待たされず（目安1秒以内）に完了すること。
- **拡張性:** 将来のログイン／サーバ同期導入時に、既存のローカルデータ構造を大きく壊さずに移行できる設計の余地を残すこと（実装方法は Generator の判断に委ねる）。
- **使いやすさ:** アカウント登録なしで起動直後から利用を開始できること。タップ操作中心で、スマホ画面（縦持ち）に最適化されていること。
- **堅牢性:** 不正な入力（空・0・非数値の金額、メンバー未選択など）に対してエラーを表示し、データの整合性を壊さないこと。
