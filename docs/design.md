# デザイン仕様: かしかり.me

**確定日:** 2026-05-30
**採用案:** 案C2 - フレッシュ・カジュアル（絵文字なし・クリーン版）
**ベース案:** 案C（フレッシュ・カジュアル）
**ソースモックアップ:** `.aidesigner/proposal-C2.html`
**実装前提:** React Native / Expo（iPhone 縦持ち専用、`StyleSheet` で実装）

---

## 0. デザインコンセプト

コーラル × ティールの明るくカラフルな配色をベースにした、親しみやすく楽しいトーン。
丸みのある書体（Baloo 2 を数字・英字に、Noto Sans JP を日本語に使用）と、カラフルなグラデーションのアイコン・アバターで、学生・若い世代の飲み会／旅行グループでも気軽に使える雰囲気を作る。

絵文字は **一切使わない**。代わりに以下で視覚的な楽しさを表現する：
- グラデーションの角丸スクエアアイコン（白抜きの太字イニシャル・漢字1文字）
- カラフルなイニシャルアバター（名前の頭文字）
- コーラルグラデーションのヘッダー・ボタン

## 2. コンポーネント仕様

### 2.1 ヘッダー（ホーム用・グラデーション）

- 背景: `gradHeader`（135deg コーラルグラデーション）
- 形状: 下端のみ角丸 **26px**（`borderBottomLeftRadius / borderBottomRightRadius: 26`）、上端は角丸なし
- パディング: 上 14 / 左右 22 / 下 20（ステータスバー／セーフエリアは別途 `SafeAreaView` で確保）
- 構成:
  - 1行目: アプリ名 `appTitle`（白・Baloo 2・26px・800）
  - 2行目: サブコピー `headerSub`（白92%・12px・500）、上マージン 3px
- ホームのサブコピー文言: **「みんなの立て替え、スッキリ精算！」**

### 2.2 サブ画面ヘッダー（戻る付き）

ホーム以外の画面（グループ作成・グループ詳細・支払い追加など）のヘッダー。

- 背景: `gradHeader`（同コーラルグラデーション）、下端角丸 **26px**
- 高さ: コンテンツに応じ可変、最低パディング 上14 / 左右20 / 下18
- 左に **戻るボタン**（`<` chevron アイコン、白・サイズ24px、タップ領域 44x44px）
- 中央または左揃えに `screenTitle`（白・22px・800・Baloo 2）
- 右に必要なら **アクションテキストボタン**（例「保存」「編集」）: 白・14px・700
- アイコンは `@expo/vector-icons`（Ionicons / Feather）を使用。chevron は `chevron-back`。

### 2.3 グループカード

- 背景: `surface` (#FFFFFF)、角丸 **22px**、パディング 18px、`cardShadow`、下マージン 16px
- 全体タップで詳細画面へ遷移（`TouchableOpacity` / `Pressable`、`activeOpacity` 0.85）
- 上段（`card-top`、横並び・縦中央・gap 13px）:
  - 左: **アイコンタイル**（後述、48x48px）— グループ名の先頭1文字を白抜き表示
  - 中央（flex:1）:
    - `cardTitle`: グループ名（16px・800）
    - `cardSub`: 「{N}人 のメンバー」（12px・500・textSub）、上マージン 2px
  - 右: **アバタースタック**（後述、最大4名 + 超過は「+N」）
- 下段（`card-bottom`、上マージン 16px、両端揃え・下揃え）:
  - 左: `totalLabel`「総支出」 + その下に `amountLarge` 金額（¥カンマ区切り）
  - 右: **ピル**「1人 ¥{平均}」（後述）

### 2.4 アイコンタイル（グループ／カテゴリアイコン）

- サイズ: 48x48px、角丸 16px、中央寄せ
- 背景グラデーション: グループごとに色をローテーション（`gradCoral` → `gradTeal` → `gradPurple` → `gradYellow` の順で割り当て、グループ作成順 index % 4）
- 中身（glyph）: グループ名の **先頭1文字**（漢字・かな・英字いずれも可）、Baloo 2・20px・800・色 `#FFFFFF`
  - ただし黄タイル（`gradYellow`）のときの文字色は `#7A5B00`（視認性確保）
- 小サイズ版（精算リストのカテゴリ等で使う場合）は 40x40px / 角丸 14px / glyph 18px

### 2.5 アバター（イニシャルアバター）

- 単体サイズ: 26x26px（カード内）、円形（角丸 999）、白枠 2px（`borderColor: surface`）
- 文字: メンバー名の先頭1文字、11px・700・白
- 背景色: メンバーの index でローテーション
  - `a1` = `#FF6B6B`（コーラル）/ 文字白
  - `a2` = `#4ECDC4`（ティール）/ 文字白
  - `a3` = `#FFD166`（黄）/ 文字 `#7A5B00`
  - `a4` = `#A66CFF`（紫）/ 文字白
  - 5人目以降は上記4色を繰り返す
- **スタック表示**: 横並びで重ねる。2個目以降は `marginLeft: -8`。最大 **4個** まで表示し、5人以上のときは4個目を「+N」アバター（背景 `#8A8A9C`・文字白）に置き換える。
- 詳細画面・支払い選択用の大きいアバターは 40x40px / 文字 16px を使用。

### 2.6 ピル（バッジ）

- ティール系（デフォルト・1人あたり平均など）:
  - 背景 `#D8F7F4`、文字色 `#0C8C83`、11px・700、パディング 縦6/横11、角丸 999
- 用途別バリエーション:
  - 「精算済み」ピル: 背景 `#D8F7F4` / 文字 `#0C8C83`（チェックアイコン付き）
  - カテゴリピル（支払い一覧）: カテゴリ色の薄い背景 + 濃い文字（後述カテゴリ色表参照）

### 2.7 プライマリボタン / FAB

- **下部固定 FAB**（ホーム・グループ詳細支払いタブ）:
  - 位置: 画面下部に固定、左右 20px のマージン、下 30px（セーフエリア考慮で `bottom: insets.bottom + 12` 程度）
  - 横幅: 左右いっぱい（左右マージン 20px を除く全幅）
  - 高さ: 56px、角丸 20px
  - 背景: `gradButton`（90deg コーラル→オレンジ）
  - `fabShadow`
  - 内容: 「+」プラス記号（22px・Baloo 2）+ ラベル（`buttonText` 16px・800・白）、gap 8px、中央寄せ
  - ホーム文言「新規グループ」、支払いタブ文言「支払いを追加」
- **フォーム保存ボタン**（作成・編集・支払い画面の最下部）:
  - 同じ `gradButton`、高さ 56px、角丸 20px、全幅、`buttonText`
- **セカンダリボタン**（キャンセル等）:
  - 背景 透明、枠線 1.5px `#F0E6DC`、文字色 `#8A8A9C`・16px・700、高さ 52px、角丸 18px
- **危険ボタン**（削除）:
  - 背景 透明、文字色 `#FF4D4F`・16px・700。または背景 `#FFECEC` / 文字 `#FF4D4F`。高さ 52px・角丸 18px。
- 押下時: `activeOpacity` 0.85（不透明度で表現。グラデーションのまま）

### 2.8 入力フォーム

- 入力欄: 背景 `surface` (#FFFFFF)、枠線 1.5px `#F0E6DC`、角丸 16px、高さ 52px、内側パディング 横16px、`inputText`（16px）
- ラベル: 入力欄の上に `inputLabel`（12px・700・textSub）、下マージン 8px
- フォーカス時: 枠線色を `#FF6B6B`（primary）に変更、枠線 1.5px のまま
- プレースホルダー色: `#8A8A9C`（textSub）
- エラー時: 枠線色を `#FF4D4F`、入力欄下に `errorText`（12px・error）を 6px マージンで表示
- 金額入力欄: 右寄せ or 大きめ表示。支払い追加画面では `amountHero`（34px・Baloo 2）で中央表示し、先頭に「¥」を付与。数字キーボード（`keyboardType="number-pad"`）。

### 2.9 タブバー（グループ詳細のセグメント切替）

支払い／精算の2タブ切替。トップのセグメントコントロール方式。

- コンテナ: 背景 `#FFFFFF`、角丸 14px、パディング 4px、横並び、`cardShadow` 軽め
- 各セグメント: flex:1、高さ 40px、中央寄せ、角丸 12px
  - 選択中: 背景 `gradButton`（または primary ソリッド `#FF6B6B`）、文字 白・14px・800
  - 非選択: 背景 透明、文字 `#8A8A9C`・14px・700
- 配置: グループ詳細ヘッダー直下、横マージン 20px、上マージン 16px

### 2.10 リスト行（支払い行・精算行）

- 支払い行: カード型（`surface`・角丸 16px・パディング 14px・下マージン 12px・`cardShadow` 軽め）
  - 左: カテゴリのアイコンタイル（40x40px、カテゴリ色グラデ）
  - 中央: 上に用途メモ（`body` 14px・700）、下に「{支払者}が立替・{割り勘人数}人で割り勘」（`caption` 11px・textSub）
  - 右: 金額（`amountLarge` 縮小版 18px・Baloo 2・800）+ 下にカテゴリピル（任意）
- 精算行（「AがBに ¥X 支払う」）: 後述「精算タブ」参照

### 2.11 セクションラベル

- `sectionLabel`（13px・800・text）、下マージン 16px。例:「グループ」「支払い履歴」「残高」「精算案」

## 3. 画面仕様

> 全画面共通: 背景 `#FFF8F2`、`SafeAreaView` でノッチ／ホームインジケータを確保。スクロール領域の下パディングは FAB のある画面で 120px。

### 3.1 グループ一覧画面（ホーム）— Sprint 1

**目的:** 作成済みグループを一覧表示し、新規作成への導線を提供する。

**レイアウト:**
1. ヘッダー（2.1 ホーム用グラデーションヘッダー）: アプリ名「かしかり.me」+ サブコピー
2. スクロール領域（上下パディング 20 / 下 120、左右 20）:
   - `sectionLabel`「グループ」
   - グループカード（2.3）を縦に並べる。各カードに: アイコンタイル / グループ名 / 「{N}人 のメンバー」/ アバタースタック / 「総支出 ¥{合計}」/ ピル「1人 ¥{平均}」
3. 下部固定 FAB「+ 新規グループ」（2.7）→ グループ作成画面へ遷移

**インタラクション:**
- カードタップ → グループ詳細画面（支払いタブ）へ
- FAB タップ → グループ作成画面へ

**空状態（グループ0件）:**
- スクロール中央に空状態ブロック（上マージン 30px、中央寄せ、左右パディング 14px）:
  - アイコン: 108x108px、角丸 34px、背景 `gradEmptyIcon`、`emptyIconShadow`。中央に 54x54px の白い円、その中に「¥」（Baloo 2・26px・800・primary `#FF6B6B`）
  - 見出し `h2`: 「まだグループがないよ！」（19px・800・text、下マージン 8px）
  - 説明文: 「旅行やイベントごとにグループを作って、立て替えをサクッと記録しよう」（13px・500・textSub・行間1.7・中央寄せ）
- FAB は空状態でも表示する。

### 3.2 グループ作成・編集画面 — Sprint 1

**目的:** グループ名とメンバーを入力して保存する。新規・編集兼用。

**レイアウト:**
1. サブ画面ヘッダー（2.2）: 左に戻る（×またはchevron）、タイトル「新規グループ」/ 編集時「グループを編集」、右に「保存」テキストボタン（白・14px・700）
2. スクロール領域（左右 20、上 20）:
   - `inputLabel`「グループ名」+ 入力欄（2.8）。プレースホルダー「例: 北海道旅行 2026」
   - `inputLabel`「メンバー」+ メンバー入力リスト:
     - 各メンバー行: イニシャルアバター（小・色ローテーション）+ 名前入力欄（flex:1、枠線スタイル）+ 右に削除アイコン（`trash` or `close`、`#8A8A9C`、タップで該当行削除）
     - 行間マージン 12px
   - 「+ メンバーを追加」ボタン（セカンダリスタイル/ 破線枠 or テキストリンク、文字色 `#FF6B6B`・14px・700）→ 空のメンバー入力行を追加
3. 最下部:
   - 保存ボタン（2.7 フォーム保存・コーラルグラデ・全幅）
   - 編集時のみその下に「グループを削除」危険ボタン（2.7）

**バリデーション／エラー:**
- グループ名が空 → 保存不可。入力欄を error 枠（`#FF4D4F`）にし、下に「グループ名を入力してください」を表示。
- メンバーが2名未満 → 保存時に「メンバーを2人以上追加してください」をエラー表示（仕様の受け入れ基準: 2名以上）。
- 名前が空のメンバー行は保存時に無視 or エラー。空行のままは保存対象から除外する方針。

### 3.3 グループ詳細画面（タブ構成）— Sprint 2 / 3

**目的:** グループの支払い履歴と精算結果を、支払い／精算の2タブで提供する。

**共通ヘッダー部:**
1. サブ画面ヘッダー（2.2）: 戻る + グループ名（`screenTitle`）+ 右に「編集」テキストボタン（→グループ編集画面）
2. ヘッダー内 or 直下に **総支出サマリー**:
   - 「総支出」ラベル + 金額（白・Baloo 2・大きめ）/ 「1人あたり ¥{平均}」を併記
   - ヘッダーがコーラルグラデのため、サマリーはヘッダー内に白文字で配置するか、直下に白カードで配置（推奨: ヘッダー直下に白サマリーカード）
3. セグメントタブバー（2.9）: 「支払い」「精算」。デフォルトは「支払い」タブ。

#### 3.3a 支払いタブ — Sprint 2

- `sectionLabel`「支払い履歴」
- 支払い行リスト（2.10）を **時系列（新しい順）** で表示。各行: カテゴリアイコンタイル / 用途メモ / 「{支払者}が立替・{人数}人で割り勘」/ 金額（¥カンマ区切り）/ カテゴリピル
- 行タップ → 支払い編集画面へ
- 下部固定 FAB「+ 支払いを追加」（2.7）→ 支払い追加画面へ
- **空状態（支払い0件）:** 空状態アイコン（`gradEmptyIcon` 流用 or ティール系）+ 見出し「まだ支払いがないよ！」+ 説明「最初の立て替えを記録してみよう」。FAB は表示する。

#### 3.3b 精算タブ — Sprint 3

- **残高セクション:** `sectionLabel`「残高」
  - 各メンバー行（カード or リスト行）: 左にイニシャルアバター + 名前、右に残高金額
    - プラス（受け取る）: 文字色 `#0C8C83`（success・ティール濃）、先頭「+」、「受け取る」注記
    - マイナス（支払う）: 文字色 `#FF6B6B`（primary・コーラル）、先頭「-」、「支払う」注記
    - ゼロ: 文字色 `#8A8A9C`、「精算済み」/「±0」
  - 金額は Baloo 2・18px・800
- **精算案セクション:** `sectionLabel`「精算案」
  - 各精算行（カード型・角丸16px・パディング14px・下マージン12px）:
    - 「{A} → {B}」を視覚化: 左に A のアバター、矢印アイコン（`arrow-forward`、`#8A8A9C`）、B のアバター
    - 中央 or 行内に文言「{A}さんが {B}さんに」（`body` 14px）
    - 右に金額「¥{X,XXX}」（Baloo 2・18px・800・text）
    - 右端 or 行末に「精算済み」チェック（Sprint 4 で機能、Sprint 3 では表示のみでも可）: 未チェックは枠線丸（`#F0E6DC`）、チェック済みは `#0C8C83` 塗り + 白チェックアイコン
  - 精算済み行: 文言・金額に取り消し線、不透明度 0.5、ピル「精算済み」表示（Sprint 4）
- **1人あたり平均負担額** を残高セクション上部 or サマリーに表示（Baloo 2 金額）。
- **空状態（精算なし／支払い0件）:** 「精算する貸し借りはまだないよ」+ 説明「支払いを記録すると精算案がここに出るよ」。全員残高0のときは「みんな精算済み！」を表示。

### 3.4 支払い追加・編集画面 — Sprint 2（カテゴリ／不均等は Sprint 4）

**目的:** 金額・支払者・用途・カテゴリ・割り勘対象を入力して支払いを記録する。

**レイアウト:**
1. サブ画面ヘッダー（2.2）: 戻る + タイトル「支払いを追加」/ 編集時「支払いを編集」+ 右「保存」
2. スクロール領域（左右 20）:
   - **金額（最上部・大きく）:** 中央に「¥」+ `amountHero`（34px・Baloo 2）の金額表示。タップで数字キーボード。下に薄い区切り or 枠。プレースホルダー「0」。
   - `inputLabel`「支払った人」: メンバーを横スクロールのチップ群 or リストから単一選択。選択中チップは背景 `gradButton`（コーラル）・文字白、非選択は背景 `#FFFFFF`・枠線 `#F0E6DC`・文字 text。各チップにイニシャルアバター + 名前。
   - `inputLabel`「用途（メモ）」: テキスト入力欄（2.8）。プレースホルダー「例: 夕食代、ホテル代」
   - `inputLabel`「カテゴリ」（Sprint 4）: カテゴリチップ群（後述カテゴリ定義）から単一選択。選択中はカテゴリ色塗り。
   - `inputLabel`「割り勘の対象」:
     - 上部に「全員」トグル/チェック（オンで全メンバー対象）
     - 下に各メンバーのチェックリスト（イニシャルアバター + 名前 + 右にチェックボックス）。チェック済み: `#FF6B6B` 塗り + 白チェック。未: 枠線丸 `#F0E6DC`。
     - （Sprint 4）不均等割り勘: 各メンバー行に金額/比率の入力欄を出すモード切替。「均等」「金額指定」「比率指定」のセグメント。
3. 最下部:
   - 保存ボタン（2.7 コーラルグラデ・全幅）
   - 編集時のみ「この支払いを削除」危険ボタン

**バリデーション／エラー:**
- 金額が未入力・0・非数値 → 保存不可。金額表示を error 色にし「金額を入力してください」を表示。
- 支払者未選択 → 「支払った人を選択してください」。
- 割り勘対象が0人 → 「割り勘の対象を1人以上選択してください」。

### 3.5 精算詳細（精算タブ内）— Sprint 3 / 5

3.3b に統合。Sprint 5 では下記を追加:
- **共有ボタン:** 精算タブ最下部に「精算結果を共有」ボタン（セカンダリ or プライマリ）。タップで `Share` API を起動し、精算案をテキスト出力（「{A}さんが{B}さんに ¥X,XXX」を改行区切り + 総支出・1人あたり平均）。

## 5. ナビゲーション・画面遷移

推奨: `@react-navigation/native` の **Native Stack Navigator**（モーダル風 push 遷移）。

```
HomeStack (Stack Navigator)
├─ Home（グループ一覧）        ← ルート
├─ GroupForm（作成/編集）      ← Home の FAB / 編集ボタンから push（presentation: modal 可）
├─ GroupDetail（詳細・タブ内包）← カードタップから push
│    └─ 内部で「支払い／精算」を State でセグメント切替（別 Tab Navigator でも可）
└─ PaymentForm（支払い追加/編集）← 支払いタブ FAB / 支払い行タップから push（presentation: modal 可）
```

**遷移仕様:**
- 前進: 右からスライドイン（標準 push）。作成・追加系はモーダル（下からスライド）でも可。
- 戻る: ヘッダー左の chevron / × ボタン、または iOS スワイプバック。
- 保存成功時: 直前画面へ戻り、一覧を即時更新（状態管理で再描画）。
- 全画面でヘッダーはコーラルグラデで統一し、ブランドの一貫性を保つ。

## 6. 空状態・エラー状態の表示仕様（まとめ）

| 状況 | アイコン | 見出し | 説明文 |
|------|---------|--------|--------|
| グループ0件 | `gradEmptyIcon` + 白円「¥」 | まだグループがないよ！ | 旅行やイベントごとにグループを作って、立て替えをサクッと記録しよう |
| 支払い0件 | `gradEmptyIcon` or ティール系 | まだ支払いがないよ！ | 最初の立て替えを記録してみよう |
| 精算なし（支払いあり・全員0） | ティール系丸アイコン | みんな精算済み！ | 貸し借りはありません |
| 精算なし（支払い0件） | `gradEmptyIcon` | 精算する貸し借りはまだないよ | 支払いを記録すると精算案がここに出るよ |

**エラー状態の共通ルール:**
- 入力エラー: 該当入力欄の枠線を `#FF4D4F` に変更し、直下に `errorText`（12px・600・`#FF4D4F`）を 6px マージンで表示。
- 保存不可時: 保存ボタンは押せるが、バリデーション失敗で遷移せずエラーを表示する（ボタン自体は常時 enabled でフィードバックを返す方式）。
- 破壊的操作（グループ削除・支払い削除）: `Alert.alert` で「削除しますか？」確認 →「削除」（destructive・赤）/「キャンセル」。
- トースト／フィードバック: 保存成功は画面遷移で示す（明示トーストは任意）。

## 7. 実装ガイダンス（Generator 向け）

### 7.1 スタイル戦略
- React Native `StyleSheet.create` を基本とする。色・スペーシング・タイポは **`theme.ts`（定数オブジェクト）に集約**し、本仕様のトークン名（`primary`, `bg`, `radius.card` 等）で参照できるようにする。
- グラデーションは `expo-linear-gradient` の `<LinearGradient>` で実装（ヘッダー・ボタン・FAB・アイコンタイル・アバターは単色でも可だが、アイコンタイル／ヘッダー／ボタンはグラデ必須）。

### 7.2 推奨ライブラリ
- ナビゲーション: `@react-navigation/native` + native-stack
- グラデーション: `expo-linear-gradient`
- アイコン: `@expo/vector-icons`（Ionicons 中心。chevron-back / add / trash / arrow-forward / checkmark）
- フォント: `@expo-google-fonts/noto-sans-jp` + `@expo-google-fonts/baloo-2`（`useFonts` でロード、ロード完了まで `AppLoading`/SplashScreen）
- 永続化: `@react-native-async-storage/async-storage`（実装方式は Generator 判断、本仕様は UI のみ規定）

### 7.3 金額フォーマット
- 表示は常に「¥」+ 3桁カンマ区切り（例 `¥128,400`）。`new Intl.NumberFormat('ja-JP')` または自前整形。マイナスは「-¥1,234」、受け取りは色で表現（先頭記号は任意）。
- 金額は **Baloo 2 / 800** を優先適用。

### 7.4 避けるべきパターン
- 絵文字の使用（コンセプト上 NG。アイコンは vector-icons かイニシャル glyph で）。
- 固定 px のみのレイアウト（`flex`・`%`・`Dimensions` で iPhone 各サイズに追従させる）。横幅は端末幅に応じて伸縮させ、左右マージン 20px を維持。
- インラインの `style={{...}}` 乱用（再利用スタイルは `StyleSheet` / `theme` に集約）。

### 7.5 レスポンシブ／セーフエリア
- 対象は iPhone 縦持ち専用。`SafeAreaView`（`react-native-safe-area-context`）でノッチ・ホームインジケータを確保。
- FAB の下位置は `insets.bottom + 12` を基準にし、ホームインジケータと重ならないようにする。
- スクロール領域は `KeyboardAvoidingView`（フォーム画面）でキーボード回避。
- 想定基準幅 375px のデザインを、より広い端末では左右マージンを保ったまま全幅追従させる。
