## 改善提案（合否に影響しない・将来の任意対応）

- **サマリーの二重表示**: グループ詳細ヘッダーの「総支出 / 1人あたり」と、精算タブ内の白サマリーカードが同一情報を二重表示している。AC1 は満たすが、精算タブのみで情報が完結するため、ヘッダー側を簡素化するか役割分担を明確化すると視覚的な冗長感が減る。
- **精算済みの全クリア方針**: 支払いの追加・編集・削除時に全精算済みマークをクリアする現方針は整合性の観点で安全側。将来的に「残高が変化した送金ペアのみ無効化」へ精緻化すると UX が向上する（progress.md 記載済みの既知課題）。
- **navigator.share の経路差**: デスクトップ Chromium ではクリップボードコピー経路（検証済み）。iOS 実機・対応モバイル Web では OS 共有シートが起動する設計。AC2「テキスト形式で出力」はコピー経路で満たしていることを確認済み。
